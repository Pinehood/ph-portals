# Support

## Getting Help

If you need help using this project, start with the repository documentation:

- `README.md` for installation and usage
- `CONTRIBUTING.md` for contribution guidance
- `SECURITY.md` for responsible vulnerability reporting

## Ask a Question

Use GitHub Discussions if the repository has them enabled.

If not, open an issue only when you are reporting a bug, regression, or feature request that belongs in the tracker.

For general support requests, replace this line with your preferred contact channel:

- E-mail: `[info@pinehood.tech]`

## Reporting Bugs

Please use the issue templates and include:

- what you expected to happen
- what actually happened
- steps to reproduce
- project version
- relevant environment details

## Security Issues

Do not report security vulnerabilities in public issues.

Follow the instructions in `SECURITY.md` instead.

## Response Expectations

This project is maintained on a best-effort basis.

Response time should be at most under one full week.
